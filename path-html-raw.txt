<h2>ğŸ”§ Repair JSON! Solution for JSON Anomalies from LLMs</h2>
<p>Welcome to the json-repair, your go-to solution for fixing those pesky JSON anomalies that can sidetrack your Large
Language Model (LLM) processes. Our toolkit is designed to be the Swiss Army knife for all your JSON repair needs.</p>
<h3>ğŸ¯ Why Choose JSON Repair?</h3>
<ul>
<li>ğŸï¸ GO Compatibility: Our library ensures a seamless experience for Go developers with its excellent compatibility.</li>
<li>ğŸ”— Zero Dependencies: We've crafted a tool with zero external dependencies, keeping it lean and mean.</li>
<li>ğŸ“š Rich Test Cases: Benefit from a comprehensive suite of test cases that ensure reliability and accuracy.</li>
<li>ğŸ¤– Auto-Detection &#x26; Repair: Intelligently identifies and corrects a wide range of JSON errors, from syntax to
structural issues.</li>
<li>ğŸ“ Terminal CLI Support: The feature can also be used in the command-line and can be chained with command pipes.</li>
<li>âš™ï¸ No Anxiety About Error: json-repair always gives the string result.</li>
<li>ğŸŒ Open Source: Join a vibrant community of developers contributing to the ongoing evolution of the toolkit.</li>
</ul>
<h3>ğŸ” Supported Broken LLM JSON Output</h3>
<ul>
<li>Single quote <code>"</code></li>
<li>Line feed <code>\n</code></li>
<li>Improperly formatted JSON string <code>{"key": TRUE, "key2": FALSE, "key3": Null  </code></li>
<li>String with mixed quotes <code>{'key': 'string', 'key2': false, \"key3\": null, \"key4\": unquoted}</code></li>
<li>Unclosed array <code>[1, 2, 3, 4</code></li>
<li>Unclosed array object <code>{"employees":["John", "Anna",</code></li>
<li>Standalone left bracket <code>[</code></li>
<li>Standalone right bracket <code>]</code></li>
<li>Array with extra line breaks <code>[[1\n\n]</code></li>
<li>Incorrect key-value pair <code>{foo: [}</code></li>
<li>Correct JSON string <code>{"text": "The quick brown fox won\'t jump"}</code></li>
<li>Incorrect key-value pair <code>{"value_1": "value_2": "data"}</code></li>
<li>JSON string with comment <code>{"value_1": true, COMMENT "value_2": "data"}</code></li>
<li>JSON string with leading spaces  <code>- { "test_key": ["test_value", "test_value2"] }</code></li>
<li>String containing a link <code>{ "content": "[LINK]("https://google.com")" }</code></li>
<li>Unclosed link string <code>{ "content": "[LINK](" }</code></li>
<li>Unclosed link and extra key string <code>{ "content": "[LINK](", "key": true }</code></li>
<li>Incorrect key-value pair <code>{"key":"",}</code></li>
<li>etc.</li>
</ul>

<h2>ğŸ Getting Started</h2>
<p>To add the JSON Repair to your Go project, use the following command:</p>
<pre><code class="language-shell">go get github.com/RealAlexandreAI/json-repair
</code></pre>
&#x3C;!-- USAGE EXAMPLES -->
<h2>Usage</h2>
<pre><code class="language-go">package main

import (
    "github.com/RealAlexandreAI/json-repair"
)

func main() {
    // broken JSON string from LLM
    in := "```json {'employees':['John', 'Anna', ```"

    jsonrepair.RepairJSON(in)

    // output:	{"employees":["John","Anna"]}
}
</code></pre>
<blockquote>
<p>Additionally, there is <code>MustRepairJSON</code> for scenarios that are not suitable for error handling, such as pipes and
trusted environments</p>
</blockquote>

the <a href="https://github.com/RealAlexandreAI/json-repair/blob/master/main_test.go">Test Cases</a>

<h2>Terminal CLI</h2>
<pre><code class="language-bash">
brew install realalexandreai/tap-jsonrepair/jsonrepair

# from raw string
jsonrepair -i "{'employees':['John', 'Anna', "
# output: {"employees":["John", "Anna", "Peter"]}

# from file
jsonrepair -f &#x3C;json-file>.json
</code></pre>
